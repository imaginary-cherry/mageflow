1. From the `frontend/` directory, install Tauri CLI and JS dependencies:
   ```
   npm install -D @tauri-apps/cli@latest
   npm install @tauri-apps/plugin-shell @tauri-apps/plugin-store @tauri-apps/api
   ```

2. Run `npx tauri init` with these answers:
   - App name: "Mageflow Viewer"
   - Window title: "Mageflow Viewer"
   - Frontend dist: "../dist"
   - Dev URL: "http://localhost:8080"
   - Before dev command: "npm run dev"
   - Before build command: "npm run build"

   If `tauri init` requires interactive input, create the files manually instead using the research patterns.

3. After init, edit `src-tauri/tauri.conf.json` to add:
   - `"identifier": "dev.mageflow.viewer"`
   - CSP: `"default-src 'self'; connect-src ipc: http://ipc.localhost http://127.0.0.1:*; img-src 'self' data: blob:; style-src 'unsafe-inline' 'self'"`
   - Bundle externalBin: `["binaries/mageflow-server"]`
   - Window size: width 1200, height 800

4. Edit `src-tauri/Cargo.toml` to add dependencies:
   - `tauri-plugin-shell` (with `"open-api"` feature)
   - `tauri-plugin-store`

5. Edit `src-tauri/src/lib.rs` to register plugins:
   ```rust
   tauri::Builder::default()
       .plugin(tauri_plugin_shell::init())
       .plugin(tauri_plugin_store::Builder::new().build())
       .run(tauri::generate_context!())
       .expect("error while running tauri application");
   ```
   Keep lib.rs minimal — sidecar lifecycle is added in Plan 02.

6. Create `src-tauri/capabilities/default.json` with permissions:
   - `core:default`
   - `store:allow-load`, `store:allow-get`, `store:allow-set`, `store:allow-save`
   - `shell:allow-execute` with sidecar config for `binaries/mageflow-server` accepting `--port`, `--host`, `--hatchet-api-key`, `--redis-url` args (see research Pattern for exact validator regexes)

7. Create `src-tauri/binaries/` directory with a `.gitkeep` file (actual binary comes in Plan 06).

8. Add `"tauri": "tauri"` to frontend/package.json scripts section.

<task type="auto">
  <name>Task 2: Update Vite config for Tauri and create Python sidecar entry point</name>
  <files>frontend/vite.config.ts, libs/mage-voyance/visualizer/__main__.py</files>
  <action>
1. Replace `frontend/vite.config.ts` with Tauri-compatible configuration:
   - Remove `lovable-tagger` plugin (dev-only tool, not needed in Tauri)
   - Remove the proxy config for `/api` (sidecar will be accessed directly via port)
   - Add `clearScreen: false` (Tauri CLI manages output)
   - Set `server.host` to `process.env.TAURI_DEV_HOST || "::"`
   - Set `server.port: 8080`, `server.strictPort: true`
   - Set HMR: `host ? { protocol: 'ws', host, port: 1421 } : { overlay: false }`
   - Add watch ignore: `['**/src-tauri/**']`
   - Add `envPrefix: ['VITE_', 'TAURI_ENV_*']`
   - Add build target: `process.env.TAURI_ENV_PLATFORM === 'windows' ? 'chrome105' : 'safari13'`
   - Add build minify: `!process.env.TAURI_ENV_DEBUG ? 'esbuild' : false`
   - Add build sourcemap: `!!process.env.TAURI_ENV_DEBUG`
   - Keep the `@` alias and `react()` plugin

2. Create `libs/mage-voyance/visualizer/__main__.py`:
   - Import argparse, os, uvicorn
   - Import `from multiprocessing import freeze_support`
   - Import `from visualizer.server import create_dev_app` (use dev app since Tauri serves the frontend — server only needs API routes with CORS)
   - Parse CLI args: `--port` (int, default 8000), `--host` (str, default "127.0.0.1"), `--hatchet-api-key` (str, default ""), `--redis-url` (str, default "redis://localhost:6379")
   - Set `os.environ["REDIS_URL"] = args.redis_url` before creating app
   - If hatchet-api-key provided, set `os.environ["HATCHET_API_KEY"] = args.hatchet_api_key`
   - Call `freeze_support()` at top of main()
   - Create app via `create_dev_app()` (not `create_app()` — the latter serves static files and has SPA fallback, which is not needed when Tauri serves the frontend)
   - Run with `uvicorn.run(app, host=args.host, port=args.port, workers=1)` — workers=1 is CRITICAL for Nuitka compatibility
   - Guard with `if __name__ == "__main__": main()`
  </action>
