  <action>
Replace the minimal `lib.rs` from Plan 01 with full sidecar lifecycle management:

1. **State structs:**
   - `SidecarState(Mutex<Option<CommandChild>>)` — holds the child process handle
   - `SidecarPort(Mutex<Option<u16>>)` — holds the allocated port (Option because it may not be ready yet)

2. **`find_free_port()` function:**
   - Use `std::net::TcpListener::bind("127.0.0.1:0")` to get OS-assigned port
   - Extract port from `listener.local_addr().unwrap().port()`
   - Drop the listener before returning (sidecar needs to bind it)
   - Return `u16`

3. **`spawn_sidecar(app: &AppHandle)` async function:**
   - Call `find_free_port()` to get a port
   - Store port in `SidecarPort` state
   - Read credentials from tauri-plugin-store: load `settings.json`, get `hatchetApiKey` and `redisUrl`
   - Use `app.shell().sidecar("binaries/mageflow-server")` to create the command
   - Pass args: `["--port", &port.to_string(), "--host", "127.0.0.1", "--hatchet-api-key", &hatchet_key, "--redis-url", &redis_url]`
   - Call `.spawn()` on the command
   - Store the `CommandChild` in `SidecarState`
   - Log spawn success/failure to stdout (for debugging)
   - If settings are not found (first launch), still store the port but do NOT spawn the sidecar — the frontend will show onboarding first, and sidecar will be spawned after settings are saved

4. **`kill_sidecar(app: &AppHandle)` sync function:**
   - Lock `SidecarState` mutex
   - If `Some(child)`, call `child.kill()` and replace with `None`
   - Log kill to stdout

5. **Tauri commands:**
   - `get_sidecar_port(state: State<SidecarPort>) -> Result<u16, String>`: Returns the port or an error string if not yet assigned
   - `get_sidecar_status(state: State<SidecarState>) -> String`: Returns "running" if child exists, "stopped" otherwise
   - `restart_sidecar(app: AppHandle) -> Result<u16, String>`: Kills existing sidecar, spawns a new one (reads fresh settings from store), returns new port. This is needed for the retry button on sidecar crash.

6. **App builder:**
   ```rust
   tauri::Builder::default()
       .plugin(tauri_plugin_shell::init())
       .plugin(tauri_plugin_store::Builder::new().build())
       .manage(SidecarState(Mutex::new(None)))
       .manage(SidecarPort(Mutex::new(None)))
       .invoke_handler(tauri::generate_handler![
           get_sidecar_port,
           get_sidecar_status,
           restart_sidecar,
       ])
       .build(tauri::generate_context!())
       .expect("error while building tauri application")
       .run(|app_handle, event| match event {
           RunEvent::Ready => {
               let app = app_handle.clone();
               tauri::async_runtime::spawn(async move {
                   spawn_sidecar(&app).await;
               });
           }
           RunEvent::ExitRequested { .. } | RunEvent::Exit => {
               kill_sidecar(app_handle);
           }
           _ => {}
       });
   ```

7. **Important notes:**
   - Use `tauri_plugin_shell::ShellExt` for `app.shell()` access
   - Use `tauri::Manager` for `app.state()` access
   - The `kill_sidecar` function must be synchronous (no async in exit handler)
   - Handle the case where settings.json doesn't exist (first launch) gracefully — store port but skip spawn
   - For reading store in Rust, use `tauri_plugin_store::StoreExt` trait: `app.store("settings.json")`

8. **Cargo.toml additions (if not already present):**
   - `serde` with `derive` feature
   - `serde_json`
