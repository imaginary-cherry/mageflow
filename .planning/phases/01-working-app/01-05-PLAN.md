  <action>
**Create `frontend/src-tauri/src/tray.rs`:**

1. **`create_tray(app: &AppHandle) -> tauri::Result<TrayIcon>`:**
   - Create menu items: "Show Window", "Settings...", "Quit"
   - Build menu with `Menu::with_items(app, &[&show, &settings, &quit])`
   - Build tray icon:
     - Start with yellow icon (starting state): load from `include_bytes!("../icons/tray-yellow.png")` using `tauri::image::Image::from_bytes()`
     - Set `menu_on_left_click(false)` — left click toggles window, not menu
     - `on_tray_icon_event`: On left click (`TrayIconEvent::Click` with `button: MouseButton::Left`), toggle the main window visibility. Get window via `tray.app_handle().get_webview_window("main")`, call `window.show()` or `window.hide()` based on `window.is_visible()`
     - `on_menu_event`:
       - "show": show and focus the main window
       - "settings": emit event to frontend: `app.emit("open-settings", ())` — the React side listens and opens SettingsDialog
       - "quit": call `app.exit(0)` — this triggers RunEvent::Exit which kills the sidecar
   - Return the built TrayIcon

2. **`update_tray_icon(app: &AppHandle, status: &str)`:**
   - Match status:
     - "connected" -> load tray-green.png
     - "starting" | "partial" -> load tray-yellow.png
     - "disconnected" | _ -> load tray-red.png
   - Get the tray icon via `app.tray_by_id("main")` or store tray handle in state
   - Call `tray.set_icon(Some(icon))`
   - Also update tooltip: `tray.set_tooltip(Some(&format!("Mageflow Viewer - {}", status)))`

3. **Tauri command `set_tray_status`:**
   ```rust
   #[tauri::command]
   fn set_tray_status(app: AppHandle, status: String) {
       update_tray_icon(&app, &status);
   }
   ```

**Update `frontend/src-tauri/src/lib.rs`:**

1. Add `mod tray;` declaration
2. Call `tray::create_tray()` in a `.setup()` hook on the Builder (before `.build()`):
   ```rust
   .setup(|app| {
       tray::create_tray(app.handle())?;
       Ok(())
   })
   ```
3. Add `set_tray_status` to the `invoke_handler` generate_handler list
4. Enable the `tray-icon` feature in Cargo.toml for the `tauri` dependency if not already present

**Important notes:**
- On Linux, `TrayIconEvent::Click` is unsupported. The right-click context menu still works. Do NOT add workarounds that break macOS/Windows.
- Per user decision: "Close button (X) quits the app and kills the sidecar — no minimize-to-tray." This means the default window close behavior is fine — we do NOT need to intercept `CloseRequested` to minimize. Let the window close, which triggers app exit, which kills the sidecar via the RunEvent::Exit handler.
- The frontend will call `invoke('set_tray_status', { status: 'connected' })` from the useAppStartup hook when phase transitions occur.
